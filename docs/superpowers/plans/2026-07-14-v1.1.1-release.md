# MacFanControl v1.1.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Commit the completed Helper and SMC fixes, publish them on `main`, and release an Apple-notarized `MacFanControl v1.1.1` with reproducible ZIP and SHA-256 assets.

**Architecture:** Keep the existing SwiftPM module boundaries and release scripts. Commit the already-tested runtime fixes separately from release metadata, fast-forward `main`, build signed Universal Binary app and Helper executables, submit the staged app to Apple notarization, and publish only the stapled artifact.

**Tech Stack:** Swift 5.9, SwiftUI/AppKit, IOKit AppleSMC, SMAppService, NSXPCConnection, SwiftPM, Developer ID codesigning, `notarytool`, `stapler`, Git, and GitHub CLI.

## Global Constraints

- Release version, Git tag, and GitHub Release are exactly `1.1.1` / `v1.1.1`.
- Release assets are exactly `MacFanControl_v1.1.1.zip` and `MacFanControl_v1.1.1.zip.sha256`.
- Main app and Helper must both contain `arm64` and `x86_64` architectures.
- Signing identity is `Developer ID Application: chen ping (E7D48R2466)` and Team ID is `E7D48R2466`.
- Apple notarization must return `Accepted`; an unnotarized artifact must not be published as `v1.1.1`.
- Do not force-push, rewrite remote history, or create a pull request.
- Do not stage unrelated files.

## MacFanControl v1.1.1

本版本集中修复 Developer ID 分发后的风扇控制服务连接、M4 SMC 数据兼容和旧版 Helper 迁移问题。

- 修复已启用服务的连接重试和状态提示。
- 通过 launchd 服务标签可靠清理旧版 Helper、plist、可执行文件和 socket。

3. 首次运行时根据提示启用风扇控制服务，并在系统设置中批准后台项目。

要求 macOS 13.0 或更高版本。本发布包使用 Developer ID 签名、通过 Apple 公证并装订公证票据。
