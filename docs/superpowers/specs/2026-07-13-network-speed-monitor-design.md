# 网络上传下载速度监控设计

## 背景

应用已有两秒一次的系统监控循环，可在不增加额外定时器的情况下加入当前活跃物理网络接口的上传、下载速度。

## 目标

- 合计当前活跃物理接口的接收和发送速率。
- 自动覆盖 Wi-Fi、内置/USB/雷雳有线网卡。
- 排除回环、VPN、AWDL、桥接和其他虚拟接口。
- 接口切换、计数器回退和采样间隔异常时不产生负值或速度尖峰。
- 菜单直接显示格式化的实时下载和上传速度。

## 接口筛选

使用 `getifaddrs` 枚举接口：

1. 收集拥有 IPv4 或 IPv6 地址、名称以 `en` 开头且带有 `IFF_UP` 和 `IFF_RUNNING` 的接口名。
2. 再从这些接口的 `AF_LINK` 节点读取 `if_data.ifi_ibytes` 和 `ifi_obytes`。
3. 对所有符合条件的接口累计字节数。

该规则排除 `lo0`、`utun*`、`awdl*`、`llw*`、`bridge*` 等接口。VPN 经由物理网卡产生的底层流量仍会计入实际 `en*` 接口。

## 速率模型

在 `MacFanControlCore` 中新增：

- `NetworkTransferSnapshot`：时间、接收累计字节、发送累计字节和活动接口集合。
- `NetworkSpeed`：下载和上传的每秒字节数。

速率计算要求：

- 首次采样只建立基线，返回零速率。
- 当前接口集合与上次不同：重建基线，返回零速率。
- 时间间隔小于等于零：返回零速率。
- 任一累计计数器变小：视为重置，返回零速率。
- 正常情况下使用字节差除以真实秒数。

格式化使用十进制单位：`B/s`、`KB/s`、`MB/s`、`GB/s`，保留一位小数；小于 0 的输入按 0 处理。

## 监控接入

`NetworkMonitor` 保存上一份有效快照。`getNetworkSpeed()` 每次读取当前快照并调用纯模型计算，然后更新基线。

`FanController` 注入 `NetworkMonitor` 并增加 `networkSpeed`。现有 `updateTemperatures()` 的主线程系统统计阶段调用一次，不新增定时器。

## 界面

温度和系统状态区域新增“网络”行：

- 下载使用向下箭头和蓝色。
- 上传使用向上箭头和绿色。
- 同一行显示 `↓ 1.2 MB/s  ↑ 320.0 KB/s`。

## 测试

- 正常字节差和真实时间间隔计算。
- 首次采样、接口集合变化、计数器回退和无效时间间隔返回零。
- B/KB/MB/GB 单位格式化。

## 非目标

- 不按应用或进程拆分流量。
- 不统计累计流量或绘制历史曲线。
- 不单独显示 VPN 虚拟接口吞吐量。
- 不调用 `nettop` 或启动常驻子进程。
