# MacFanControl 区域截图与编辑功能设计

## 目标

在现有 macOS 菜单栏应用中增加区域截图工具。用户可以通过默认或自定义的全局快捷键进入截图模式，在鼠标所在显示器内框选任意大小的区域，随后进入独立编辑窗口。编辑完成后可以通过系统保存面板输出 PNG 或 JPEG，也可以复制到系统剪贴板；编辑器还支持从剪贴板粘贴图片继续编辑。

截图能力必须与现有风扇监控和自动控制隔离，不进入两秒监控循环，也不能因截图窗口、编辑缓存或事件监听产生持续 CPU 占用和内存泄漏。

## 第一版范围

第一版包含：

- 区域截图。
- 默认全局快捷键 `Control + Shift + A`。
- 在设置中录入自定义快捷键并恢复默认值。
- 在菜单栏面板中提供“区域截图”按钮。
- 鼠标所在显示器内的任意尺寸框选。
- 截图后始终打开编辑器。
- 裁剪、矩形、箭头、画笔、文字和马赛克。
- 选择、移动、调整和删除已有标注。
- 颜色、线宽和透明度调整。
- 撤销和重做。
- 使用系统保存面板保存 PNG 或 JPEG。
- 将最终图片复制到系统剪贴板。
- 在编辑器内通过 `Command + V` 打开剪贴板图片。

第一版不包含：

- 窗口智能识别。
- 全屏截图快捷键。
- 跨显示器连续框选。
- 长截图或滚动截图。
- OCR、翻译和二维码识别。
- 桌面贴图悬浮窗。
- 录屏和录音。
- 标注元素之间的复制粘贴。

## 技术方向

采用 AppKit 与 SwiftUI 混合实现：

- AppKit 负责全局快捷键、屏幕权限、显示器识别、无边框遮罩窗口、鼠标事件、屏幕坐标换算和截图会话生命周期。
- SwiftUI 负责截图编辑器、标注工具栏、菜单入口和设置界面。
- CoreGraphics 负责显示器图像捕获、选区裁剪和最终图片合成。
- AppKit 的 `NSPasteboard` 与 `NSSavePanel` 分别负责剪贴板和文件输出。

该方向保留现有项目的 SwiftUI 结构，同时使用 AppKit 处理 SwiftUI 不擅长的屏幕级窗口和输入事件。

## 组件边界

### `GlobalHotKeyManager`

负责注册、注销和更新截图快捷键。它从持久化设置加载快捷键，并在用户修改时先验证组合是否合法、是否注册成功。注册失败或冲突时保留上一个有效快捷键并向用户显示错误。

快捷键只触发截图协调器，不直接创建窗口或读取屏幕。

### `ScreenCaptureCoordinator`

负责一次截图会话的完整生命周期：

1. 防止同时启动多个会话。
2. 检查屏幕录制权限。
3. 根据鼠标位置确定目标显示器。
4. 捕获该显示器的原始画面。
5. 创建选区遮罩。
6. 根据选区裁剪原图。
7. 关闭遮罩并打开编辑器。
8. 在取消或失败时释放会话资源。

它不依赖 `FanController`，也不参与风扇监控计时器。

### `SelectionOverlayController`

负责目标显示器上的透明无边框窗口、截图背景、暗色遮罩、十字光标、像素放大预览、实时尺寸标签和鼠标框选。

交互规则：

- 鼠标按下确定起点，拖动更新选区，松开完成。
- 支持向任意方向拖动，内部统一规范化为正向矩形。
- 小于 `4 × 4` 逻辑点的选区视为误触并取消。
- `Esc`、右键或截图会话失去有效状态时取消。
- 同一时间只存在一个遮罩窗口。
- 选区限制在鼠标启动时所在显示器内，不能跨屏。

### `ScreenshotEditorDocument`

编辑文档保存：

- 不可变的原始图片。
- 当前裁剪范围。
- 标注元素数组。
- 当前选择状态。
- 撤销和重做历史。
- 是否存在未保存修改。

标注使用统一枚举建模，包括矩形、箭头、画笔路径、文字和马赛克区域。每个元素保存自身几何信息、颜色、线宽和透明度。标注保持非破坏式，保存或复制时才与底图合成。

### `ScreenshotEditorWindowController`

负责独立编辑窗口的创建、激活、关闭确认和文档切换。截图完成后始终打开编辑器。

窗口顶部提供选择、裁剪、矩形、箭头、画笔、文字、马赛克、撤销和重做；底部显示当前工具的颜色、线宽和透明度；右上角固定“复制”和“保存”。关闭存在未保存内容的窗口或粘贴另一张图片前必须确认。

### `ScreenshotOutputService`

负责：

- 将底图、裁剪范围和标注合成为最终 `CGImage`。
- 通过 `NSSavePanel` 保存 PNG 或 JPEG。
- 将最终图片写入 `NSPasteboard`。
- 从 `NSPasteboard` 读取可用图片并创建新编辑文档。

系统保存面板默认使用 PNG，用户可改选 JPEG。取消保存不算错误，也不会关闭编辑器。

## 截图数据流

1. 用户按下全局快捷键或点击菜单栏中的“区域截图”。
2. 协调器确认当前没有其他截图会话。
3. 调用系统屏幕捕获权限预检；未授权时显示用途说明并请求授权或打开系统设置。
4. 根据 `NSEvent.mouseLocation` 找到鼠标所在 `NSScreen` 和对应显示器标识。
5. 在显示遮罩前捕获该显示器的完整原始图像。
6. 遮罩使用已捕获图像作为背景，并在其上绘制暗层、选区和辅助信息。
7. 用户松开鼠标后，协调器将逻辑坐标转换为图像像素坐标并裁剪原图。
8. 立即关闭遮罩，释放完整显示器缓存，创建只包含选区图片的编辑文档。
9. 用户完成编辑后选择保存或复制，输出服务合成最终图片。

在遮罩显示前捕获屏幕，可以确保遮罩、尺寸标签、放大镜和鼠标光标不会进入最终图片。

## 坐标与多显示器

截图会话只作用于鼠标所在显示器。每块显示器可以独立截图，但不能跨显示器框选。

AppKit 使用以点为单位、原点方向与 CoreGraphics 不同的全局坐标；屏幕图像使用像素坐标。因此坐标转换必须显式包含：

- 目标显示器在全局桌面中的原点。
- AppKit 与图像坐标的 Y 轴翻转。
- 显示器 `backingScaleFactor`。
- 选区对图像边界的裁剪。

尺寸标签显示逻辑点尺寸，最终图片保持显示器的原始像素分辨率。

## 编辑与历史记录

除裁剪外，所有标注操作都作用于独立元素：

- 创建元素产生一条可撤销命令。
- 移动、缩放、样式修改和删除均产生可撤销命令。
- 连续拖动画笔在鼠标松开时合并为一条命令。
- 裁剪修改画布范围，并同步转换标注坐标；一次裁剪作为一条命令。
- 新操作发生后清空重做栈。

撤销历史仅存在于当前编辑会话，不写入磁盘。关闭编辑器后立即释放。

## 剪贴板行为

“复制”将当前合成结果写入系统剪贴板，使其可以直接粘贴到微信、文档和其他支持图片的应用中。

编辑器接收 `Command + V` 时读取剪贴板图片。如果当前文档存在未保存修改，先要求用户确认；确认后用剪贴板图片创建新文档。剪贴板没有图片、数据损坏或图片超过限制时，保留当前文档并显示错误。

为避免异常图片造成内存暴涨，剪贴板输入限制为最多一亿像素。超过限制时拒绝载入，并提示用户先缩小图片。

## 权限、错误和恢复

- 未取得屏幕录制权限时不创建无效遮罩，直接显示权限说明和系统设置入口。
- 权限被撤销、显示器断开或捕获失败时结束当前会话并释放资源。
- 重复触发快捷键时，如果已有截图会话则忽略重复请求。
- 编辑器已打开不妨碍开始新的截图；新截图创建新的编辑文档窗口，不覆盖已有未保存内容。
- 保存失败、编码失败和剪贴板写入失败在编辑器内显示具体错误。
- 用户取消截图或取消保存属于正常流程，不记录为应用错误。
- 截图错误不得修改风扇控制配置或停止硬件监控。

## 资源与性能

- 完整显示器图像只在选区阶段持有；裁剪成功后立即释放。
- 遮罩窗口关闭后解除事件处理闭包和控制器引用。
- 编辑器只缓存原始选区图和轻量标注模型，不持续缓存每一步的整张位图。
- 撤销操作保存模型变化，不保存完整图片副本。
- 马赛克和最终合成按需渲染；参数未变化时可复用一次合成结果，文档变更后使缓存失效。
- 关闭编辑窗口时释放底图、合成缓存和撤销历史。
- 全局快捷键监听常驻，但不进行轮询。

## 界面接入

菜单栏面板增加“区域截图”按钮，并在按钮旁显示当前快捷键。该入口与全局快捷键调用同一个截图协调器。

设置窗口增加“截图”区域，包含：

- 当前区域截图快捷键。
- 录入新组合键的控件。
- 恢复默认快捷键按钮。
- 屏幕录制权限状态和系统设置入口。

截图编辑器使用独立普通窗口，不嵌入菜单栏弹窗，以保证足够的画布空间和正确的键盘焦点。

## 测试与验收

### 自动测试

可测试核心逻辑与系统 API 分离。通过协议封装屏幕捕获、权限、剪贴板和快捷键注册，并使用模拟实现测试：

- 正向和反向拖动的选区规范化。
- 选区最小尺寸和屏幕边界限制。
- AppKit 点坐标到 Retina 图像像素坐标的换算。
- 裁剪后的图片尺寸。
- 标注创建、移动、缩放、删除和样式修改。
- 撤销与重做顺序。
- 裁剪时的标注坐标转换。
- PNG 和 JPEG 编码结果。
- 剪贴板无图片、损坏图片和超大图片处理。
- 默认快捷键、持久化、自定义和冲突回退。
- 重复截图请求只创建一个会话。
- 取消和失败后会话状态恢复为空闲。

### 手工验收

- 在单显示器和多显示器上分别截图。
- 在 Retina 显示器上确认选区与输出像素一致。
- 验证 `Esc`、右键和小区域误触取消。
- 拒绝、授予和撤销屏幕录制权限。
- 修改快捷键、制造冲突并恢复默认值。
- 使用全部六种编辑工具并进行撤销、重做和裁剪。
- 保存 PNG 和 JPEG。
- 将图片复制并粘贴到微信、文档和预览。
- 从剪贴板打开图片继续编辑。
- 连续完成多次截图，关闭编辑器后观察内存回落。
- 截图及编辑期间确认风扇监控和自动控制持续运行。

## 完成标准

当用户可以通过默认或自定义快捷键稳定完成“框选区域 → 编辑 → 保存或复制”，并且权限拒绝、取消、多显示器、Retina、剪贴板异常和资源释放均符合本设计，同时所有自动测试通过，即认为第一版完成。
