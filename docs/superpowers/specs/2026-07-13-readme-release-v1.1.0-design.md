# MacFanControl v1.1.0 README 与发布设计

## 目标

让项目首页准确反映当前应用能力，并发布包含最新截图功能、系统监控和稳定性修复的 `v1.1.0` 正式版本。

## README 内容

README 保持中文为主，突出应用是原生 macOS 菜单栏工具，并按实际代码描述以下能力：

- CPU、GPU、SSD 温度以及风扇转速监控；仅展示能够明确识别的传感器。
- CPU、内存、启动磁盘容量和当前活跃物理网络接口上传下载速度。
- 手动风扇控制、静音/平衡/性能预设、自定义曲线和自动控制配置持久化。
- 默认 `Control + Shift + A` 的可配置全局区域截图快捷键。
- 截图编辑器支持裁剪、矩形、箭头、画笔、文字、马赛克、选择移动、缩放、删除、撤销和重做。
- PNG/JPEG 保存、复制到剪贴板以及从剪贴板打开图片。
- 屏幕录制权限、特权 Helper 和应用安装的必要说明。

删除或修正与现状不符的表述，包括“20+ 个传感器”、旧测试数量、旧 target 数量以及仅能从 `.build/debug` 运行的安装说明。架构图和项目结构补充 `ScreenshotKit`、截图协调器和截图测试。

版本号在 `Sources/Info.plist`、`build-app.sh` 和应用“关于”页面保持一致。

## 构建与验证

发布前执行：

1. `swift test`，要求全部测试通过。
2. `swift build -c release`，要求编译成功。
3. `./build-app.sh` 生成 `MacFanControl.app`。
4. 校验 plist、临时签名、稳定指定要求和 arm64 主程序。
5. 使用 `ditto -c -k --sequesterRsrc --keepParent` 生成 ZIP，并用 `unzip -t` 验证。
6. 确认 Git 工作区干净、标签不存在且本地 `main` 与远端同步。

## GitHub Release 内容

发布说明按用户可见变化组织：

- 新增区域截图、编辑、保存和剪贴板工作流。
- 新增 SSD 使用量与当前活跃物理网络接口速率。
- 修复自动控制配置和自定义曲线重启后丢失。
- 修复启动后风扇控制延迟、截图权限随构建失效及编辑器窗口过小。
- 列出屏幕录制权限和管理员 Helper 安装要求。

Release 创建后再次读取远端标签、Release 页面和资产摘要，确认版本、提交、文件名和校验信息正确。
