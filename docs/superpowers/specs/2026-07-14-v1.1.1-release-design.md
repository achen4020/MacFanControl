# MacFanControl v1.1.1 发布设计

## 目标

将当前 Developer ID 分发分支中的 Helper、SMC 与风扇控制修复作为 `v1.1.1` 补丁版本发布。正式产物必须是同时支持 `arm64` 和 `x86_64` 的 Universal Binary，使用 Developer ID 签名、通过 Apple 公证并装订票据，再发布到 GitHub Releases。

## 发布范围

本版本只包含当前分支已经完成并验证的稳定性修复：

- 使用 `SMAppService` 管理受签名约束的特权 Helper，并通过双向签名校验的 XPC 通信控制风扇。
- 修正 Swift SMC 数据结构与 AppleSMC C ABI 的布局差异。
- 支持 Apple Silicon 风扇数据使用的 `flt ` 浮点格式，同时保留旧机型的 `fpe2` 格式。
- 保留第一次 `GetKeyInfo` 返回的 SMC 元数据，避免读取阶段丢失数据类型和长度。
- 将不支持的可选 SMC 键视为能力缺失，不影响其余风扇和温度数据。
- Apple Silicon 在 Helper 重连期间继续使用 XPC，不降级到无权限的直接 SMC 调用。
- 通过 launchd 服务标签迁移并删除旧版 `com.macfancontrol.helper` 和 `com.macfancontrol.smchelper` 服务。
- 改进 Helper 未安装、待批准、已启用但暂时不可达等状态的提示和操作入口。

不在本版本中增加新的截图、监控或风扇曲线功能。

## 版本与文档

- Git 标签与 GitHub Release：`v1.1.1`
- 应用短版本：`1.1.1`
- 应用构建版本：`1.1.1`
- 发布资产：`MacFanControl_v1.1.1.zip`
- 校验资产：`MacFanControl_v1.1.1.zip.sha256`

同步修改 `Sources/Info.plist`、设置页版本显示、README 下载和发布命令、发布元数据测试，并新增 `docs/releases/v1.1.1.md`。发布说明重点解释 Helper 安全架构、M4 SMC 兼容和旧服务迁移修复。

## 集成与提交

当前功能分支为 `codex/developer-id-distribution`，基础分支为 `main`。提交顺序为：

1. 提交本发布设计。
2. 提交已完成的 Helper、SMC 和风扇控制修复。
3. 提交 `v1.1.1` 版本元数据、README 和发布说明。
4. 将功能分支合并到本地 `main`，在合并结果上重新运行完整验证。
5. 推送 `main`，在已推送的发布提交上创建 `v1.1.1` 标签和 GitHub Release。

不会创建额外 PR，也不会强制推送或重写远端历史。

## 构建、公证与资产

1. 运行完整 Swift 测试和 shell 发布合约测试。
2. 使用 `scripts/build-developer-id-release.sh` 分别构建 `arm64` 与 `x86_64` 主应用和 Helper，并合并为 Universal Binary。
3. 按 Helper、主应用的顺序使用 `Developer ID Application: chen ping (E7D48R2466)` 签名，开启 Hardened Runtime 和安全时间戳。
4. 使用钥匙串中的 `notarytool` 配置提交 Apple 公证并等待 `Accepted`。
5. 将公证票据装订到暂存 App，执行 `stapler validate` 与 `spctl --assess`。
6. 生成最终 ZIP 和 SHA-256 文件，并验证 ZIP 可完整解压。

如果没有可用的 `notarytool` 钥匙串配置，或 Apple 公证未返回 `Accepted`，发布流程必须停止，不能上传未公证资产作为正式 `v1.1.1`。

## GitHub Release

Release 标题为 `MacFanControl v1.1.1`，使用 `docs/releases/v1.1.1.md` 作为说明，并上传 ZIP 与 SHA-256 两个资产。Release 必须满足：

- 标签指向已验证并已推送的 `main` 发布提交。
- `isDraft=false` 且 `isPrerelease=false`。
- 资产名称、应用版本和 README 下载说明一致。
- Release 页面和远端标签均可读取。

## 验证标准

发布前必须确认：

- `swift test` 全部通过。
- Developer ID、发布元数据、签名和公证脚本合约测试全部通过。
- 主应用和 Helper 均为 `arm64 x86_64` Universal Binary。
- `codesign --verify --deep --strict` 通过。
- `stapler validate` 和 `spctl --assess --type execute` 通过。
- 实机 XPC 可以读取风扇、设置目标转速并恢复自动模式。
- 旧版 launchd 服务、plist、可执行文件和 socket 均不存在。
- 最终 ZIP 校验通过，SHA-256 与上传资产一致。
- GitHub Release 标签、目标提交、发布状态和资产列表正确。

## 失败处理

- 测试、签名、公证、装订或 Gatekeeper 校验任一失败时停止发布。
- 发布资产只从已验证的暂存 App 生成，失败时不覆盖已有正式资产。
- GitHub Release 创建失败时保留本地标签和资产，先确认远端状态再重试，避免重复 Release。
