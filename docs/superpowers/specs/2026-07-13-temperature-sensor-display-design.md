# 温度传感器名称与过滤设计

## 背景

Apple Silicon 的私有 HID 接口在当前 Mac16,10 上返回 40 个温度通道。现有代码把 `PMU2 tdie*` 推测为效率核心、把 `tdev*` 推测为芯片区域，但缺少可靠的硬件对应依据，容易向用户展示误导性名称。

## 目标

- “更多传感器”只显示能够可靠解释的温度通道。
- 名称表达传感器通道，而不把私有通道推测为具体物理核心。
- 未知、校准和无法明确对应硬件部件的通道不进入列表及数量统计。

## 白名单规则

- `PMU tdie<数字>`：显示为 `CPU 温度通道 <数字>`。
- `NAND CH0 temp`：显示为 `SSD`。
- `PMU2 tdie*`、`PMU tdev*`、`PMU2 tdev*`、`PMU tcal`、`PMU2 tcal`：隐藏。
- 其他未知名称：隐藏。

匹配必须覆盖完整名称，避免包含相似片段的未知传感器被误收录。CPU 通道按数字自然排序，SSD 排在 CPU 通道之后。

## 模型与界面

`TemperatureInfo` 提供可选的 `displayName` 和可展示判断。无法识别时返回 `nil`，由模型层统一表达“不可展示”，避免界面重复维护规则。

`TemperatureSection` 先生成过滤、排序后的可展示列表，再用该列表控制：

- 是否显示“更多传感器”按钮。
- 按钮中的传感器数量。
- 展开后的传感器行。
- 超过 12 项时的剩余数量。

主界面的 CPU、最高温度和 GPU 汇总不受影响。

## 测试

- `PMU tdie1` 和多位编号能得到清晰名称。
- `NAND CH0 temp` 显示为 SSD。
- `PMU2 tdie*`、`tdev*`、`tcal` 和未知名称返回不可展示。
- 可展示传感器使用自然数字顺序排序。

## 非目标

- 不推测 GPU、内存或具体性能/效率核心对应关系。
- 不改变 HID 传感器读取方式。
- 不改变 CPU 温度和最高温度的计算方式。
